## 诊断并修复 /account/api/test 500 错误日志缺失问题

- Cloudflare Worker 日志显示 `POST /account/api/test` 返回 500，但日志中没有实际错误信息（`cpuTimeMs: 4, wallTimeMs: 768`）
- **分析**：最可能原因是 `forwardChatCompletion` 中的 `fetch()` 调用发生网络级异常（DNS/连接超时/TLS），抛出非 `ApiError` 被 `toApiError` 归类为 500
- **根本问题**：`src/index.ts` fetch 函数的 catch 块没有 `console.error` 记录实际错误，导致 Worker 日志中看不到真因
- **修复**：在 catch 块中加入 `console.error`，记录 `[requestId] METHOD path: error`，方便后续排查
- 类型检查通过

## 修复 POST /v1/tasks 500 错误 — 任务已创建但被入队/用量记录失败掩盖

- Cloudflare Worker 日志显示 `POST /v1/tasks` 返回 500（`cpuTimeMs: 7, wallTimeMs: 547`）
- 分析执行链路：`handleCreateTask` → `saveTask`(D1写入) → `enqueueCreatedTask`(Queue发送) → `recordTaskUsage`(用量写入)
- **Bug 1**：`enqueueCreatedTask` 在 Queue 发送失败后 `throw err`，导致已保存到 D1 的任务被 500 掩盖，且永远无法被处理。修复：改为记录日志后 return，不抛异常
- **Bug 2**：`recordTaskUsage` 在任务创建成功后抛出会返回 500，掩盖实际已成功的任务。修复：用 try-catch 包裹，失败仅记录日志
- **同类型修复**：`account.ts` 中 `enqueueAccountTestTask` 同样 `throw err` 问题一并修复
- 类型检查通过
