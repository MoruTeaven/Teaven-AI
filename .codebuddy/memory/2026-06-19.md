# 2026-06-19

## moark-async: 修复 root domain base_url 导致 404 的问题
- **问题**: 用户 credential 中 `base_url` 填的是 `https://ai.gitee.com/`（根域名），缺少 `/api/v1` 路径，导致请求 `https://ai.gitee.com/image_generation` 返回 404（Next.js 报错页面）
- **修复**: 在 `normalizeBaseUrl` 中增加对 `https://ai.gitee.com` 根域名的检测，自动补全为 `https://ai.gitee.com/api/v1`
- **文件**: `src/providers/moark-async.ts`

## moark-async: Gitee AI API 路径更新
- **问题**: Gitee AI 的 API 地址已变更，旧路径 `/image_generation` 返回 404
- **正确 API**: `POST https://ai.gitee.com/v1/async/images/generations`
- **修改内容**:
  - `DEFAULT_BASE_URL`: `https://ai.gitee.com/api/v1` → `https://ai.gitee.com/v1`
  - `DEFAULT_CREATE_PATH`: `/image_generation` → `/async/images/generations`
  - `DEFAULT_POLL_PATH`: `/task/{task_id}` → `/async/images/generations/{task_id}`
  - `LEGACY_BASE_URL` 改为 `LEGACY_BASE_URLS` (Set)，包含旧版 `api.gitee.com/v1` 和 `ai.gitee.com/api/v1`
  - `normalizeBaseUrl` 用 Set 统一检测所有旧地址并自动迁移
  - `isCreateEndpoint` 正则更新为 `/async/images/generations`

## 任务状态链 stage 中文化
- 在 `src/routes/account.ts` 新增 `stageText()` 函数，将 21 个 stage 标识符映射为中文（如 `task.created`→`任务创建`）
- 修改 `renderTaskEvents` 函数，stage 列和 status 列均使用中文映射显示
- 保持后端 stage 字符串不变，仅在 UI 层做翻译映射

## poll 事件链增强：记录实际请求 URL 和上游原始响应
- **背景**: 轮询上游返回 404 时，事件链只显示 `http=404` 和 message，看不到具体请求的 URL 和完整响应体
- **修改文件**:
  - `src/providers/types.ts`: `TaskPollResult` 新增 `poll_url`、`upstream_raw_body` 字段
  - `src/types.ts`: `AsyncTaskEvent` 新增 `poll_url` 字段
  - `src/providers/moark-async.ts`: `pollMoarkTask` 所有返回路径携带 `pollUrl` 和上行原始响应体（截断 1000 字符）；`readMoarkErrorSummary` 也返回 `raw_body`
  - `src/tasks/processor.ts`: `poll.result` 事件中附加 `poll_url`、`details.poll_url`、`details.upstream_raw_body`；`task.failed` 事件也附带上游上下文
- **效果**: 之后 404 类错误可以直接在事件链中看到 `poll_url=https://ai.gitee.com/v1/async/images/generations/UYE3ZWZ9TXVVOXGF6ZWRKMMIPGFL271V · upstream_raw_body=<html>...</html>`
