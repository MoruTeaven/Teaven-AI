## 简化类型名称（Task Types & Capability Keys）

将所有类型名称从 dot-separated 格式简化为单级形式：

| 旧名称 | 新名称 |
|---|---|
| `image.generation` | `image` |
| `image_generation` | `image` |
| `video.generation` | `video` |
| `video_generation` | `video` |
| `file.processing` | `file` |
| `chat.completions` | `chat` |
| `images.generations` (capability key) | `image` |
| `audio.speech` | `speech` |
| `audio.transcriptions` | `transcriptions` |
| `audio.translations` | `translations` |

### 修改文件：
- `src/routes/account.ts` - taskTypeForModality + 文档/示例 HTML
- `src/routes/async-image-generations.ts` - 任务记录的 type 字段
- `src/tasks/processor.ts` - 任务类型判断 + capability 键名

注意：admin.ts 的 taskTypeText 保留了所有旧名称的兼容映射，历史数据库中的旧类型值仍可正确显示。

## 模型列表接口增加 modality 筛选

`GET /v1/models?modality=image` 可按分类筛选模型。修改文件：
- `src/routes/models.ts` - handleListModels 接受 request 参数，解析 modality 查询参数进行过滤；响应中每个 model 新增 modality 字段
- `src/index.ts` - 调用时传入 request
有效 modality 值：text、image、video、file；不传则返回全部。
