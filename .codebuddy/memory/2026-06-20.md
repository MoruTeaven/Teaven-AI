## 模型列表接口增加 modality 筛选

`GET /v1/models?modality=image` 可按分类筛选模型。修改文件：
- `src/routes/models.ts` - handleListModels 接受 request 参数，解析 modality 查询参数进行过滤；响应中每个 model 新增 modality 字段
- `src/index.ts` - 调用时传入 request
有效 modality 值：text、image、video、file；不传则返回全部。

## 模型"价格"字段支持按次/按Token计费

在价格字段基础上新增 `price_unit` 区分计费方式（`per_1m_tokens` | `per_call`）：
- `src/types.ts` - 新增 `PriceUnit` 类型；`UpstreamModelConfig` 和 `ModelConfig` 新增 `price_unit?: PriceUnit`
- `src/config.ts` - `listModels()` 传播 `price_unit`
- `src/routes/admin.ts`:
  - `normalizeModelInput()` 校验并解析 `price_unit`
  - HTML 表单新增"计费单位"下拉选择框 `#model-price-unit`
  - JS 新增 `priceText(price, unit)` 辅助函数，按单位显示"X 元 / 1M Token"或"X 元 / 次"
  - 表单提交/回填/重置/JSON 导出均包含 `price_unit`
  - 列表卡片、详情视图、上游模型表格均使用 `priceText()` 显示

`GET /v1/models` 响应中每条 model 新增 `price` 和 `price_unit` 字段：

- `src/routes/account.ts` 用户中心文档：

  - 新增"响应字段"表格说明 id/modality/price/price_unit 等字段
  - 描述文字更新，说明 price 为 null 表示未定价
