## 实现异步任务完整生命周期追踪（Queue Consumer）

### 改动的文件
- `src/providers/types.ts` — 新增 `TaskPollResult` 接口和 `ProviderAdapter.pollTask` 方法
- `src/providers/moark-async.ts` — 实现 `pollMoarkTask()` 轮询上游任务状态
- `src/routes/async-image-generations.ts` — 创建任务时存储完整 `provider_context`（base_url, credential_id, config）
- `src/routes/tasks.ts` — `handleCreateTask` 新增模型→路由→插件解析，任务创建时存储上游上下文
- `src/tasks/processor.ts` — **新建**，Queue Consumer 核心编排逻辑
- `src/index.ts` — 新增 `queue()` 消费入口
- `wrangler.toml` — 新增 `[[queues.consumers]]` 配置（max_batch_size=10, max_retries=3, DLQ）

### 完整生命周期
`POST /v1/tasks` 或 `/v1/async/images/generations`:
1. **queued** → 任务创建，入 TASK_QUEUE
2. **running** → Consumer 从队列拉取，状态变更
3. 轮询上游 → Provider.pollTask / Moark pollMoarkTask
4. **succeeded** → 处理 output，可选 store_output→R2，触发 callback_url
5. **failed** → 记录 error
6. **expired** → 超过 300 次轮询上限
7. 每次状态变更:
   - `task.updated_at` 更新
   - `GET /v1/tasks/:id` 实时可查
   - `provider_context._poll_count` 递增
   - `provider_context._last_error` 记录异常
   - 终态时 `completed_at` 写入

### Consumer 容错
- 轮询异常不立即失败，退避重试
- 上游 4xx → 永久失败；5xx → 继续轮询
- 消息处理异常后 ack 并记录日志
- DLQ 兜底

## 修复 Admin 登录 500 错误

### 问题
POST /admin/login 返回 500，Cloudflare Workers 日志无具体错误信息。

### 根因
`src/auth/admin.ts` 的 `base64UrlEncode()` 函数使用了浏览器 DOM API `btoa()`，在 Cloudflare Workers 运行时中默认不可用（wrangler.toml 未配置 `nodejs_compat`）。登录成功后 `createAdminSession` → `signAdminSession` → `base64UrlEncode` 抛出 `ReferenceError: btoa is not defined`，被 catch 捕获返回 500。

### 修复
- `src/auth/admin.ts`: 将 `base64UrlEncode` 改为纯 JS 实现，直接输出 URL-safe base64（`-_` 字符集），不依赖任何运行时 API
- `src/routes/admin.ts`: `handleAdminLogin` catch 块新增 `console.error` 日志，方便 wrangler tail 排查

## 修复用户中心"任务管理"查看详情无反应

### 问题
用户中心 → 任务管理 → 点击"查看详情"无反应，但仪表盘 → 最近任务中的"查看详情"正常。

### 根因
`renderTasks()`（仪表盘）和 `renderTaskDetail()`（任务管理）都生成 `<tr id="task-row-xxx">`。两个 section 同时存在于 DOM（一个 hidden 一个 visible）。`toggleTaskDetail()` 中 `getElementById('task-row-xxx')` 找到的是 DOM 中第一个匹配的元素——即仪表盘那行（hidden），详情行被插入到 hidden 区域，视觉上无反应。

### 修复
- `renderTaskDetail()`: 行 id 改为 `task-detail-row-` 前缀，区分于仪表盘的 `task-row-`
- `toggleTaskDetail()`: 优先查找 `task-detail-row-`，找不到再 fallback 到 `task-row-`（兼容仪表盘）
