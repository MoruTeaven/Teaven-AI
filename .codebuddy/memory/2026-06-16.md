## 修复管理后台 innerHTML null 错误

- 报错：`can't access property "innerHTML", document.getElementById(...) is null`
- 根因：`renderModels()` 函数（admin.ts:2116）中调用 `document.getElementById('model-upstreams').innerHTML = ...`，但 `ADMIN_APP_HTML` 模板中不存在 `id="model-upstreams"` 的元素（该 ID 仅存在于旧版 `ADMIN_HTML` 模板中）
- 修复：添加 null 守卫 `var modelUpstreams = document.getElementById('model-upstreams'); if (modelUpstreams) modelUpstreams.innerHTML = ...`

## 管理员后台查看/复制已创建的 API Key 明文

- 需求：允许管理员在后台查看已创建 API Key 的明文，但需先通过密码验证账号归属
- 修改文件：
  - `src/admin/store.ts`：
    - `AdminApiKey` 接口新增 `encrypted_key?` 和 `encrypted_key_iv?` 可选字段（存储 AES-GCM 加密后的 Key 明文）
    - `createAdminApiKey` 创建 Key 时同时用 AES-GCM 加密原始 token 并存储
    - 新增 `revealAdminApiKeyToken(env, apiKey)` 公开函数，解密返回原始 token
    - 新增私有函数：`encryptApiToken`/`decryptApiToken`（AES-GCM 加密/解密）、`deriveEncryptionKey`（从 ADMIN_TOKEN 通过 SHA-256 派生 256 位 AES 密钥）、`base64UrlDecodeToBuffer`
  - `src/routes/admin.ts`：
    - 新增路由 `POST /admin/api/api-keys/:id/reveal`，handler `handleRevealAdminApiKey`
    - 验证流程：查找 API Key → 从 body 取 password → `verifyAdminPassword` 验证 → 解密返回 token
    - 旧版本 Key（无 encrypted_key）会提示"无法查看明文，请重新创建"
    - 前端：每个 Key 卡片新增"查看密钥"按钮（`data-key-view`）
    - 新增密码验证模态框 `#key-reveal-modal`，包含密码输入、错误提示、结果展示（含复制按钮）
    - `handleKeyAction` 拦截 `data-key-view` 点击打开模态框
    - 新增 `revealApiKey()` 函数：调 reveal API → 展示明文 token + 复制按钮
    - 新增 `.secret` 和 `.alert` CSS 类
  - 安全性：
    - encrypted_key/iv 不会通过 `publicApiKey` 暴露到列表接口
    - 每次查看需重新输入管理员密码验证
    - 加密密钥派生自 ADMIN_TOKEN，管理员密码变更后旧 Key 不可解密
