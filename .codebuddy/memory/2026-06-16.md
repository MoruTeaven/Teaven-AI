# 2026-06-16

## 管理员一键登录其它用户（Impersonate 功能）

- 在 `src/routes/account.ts` 中导出了 `serializeAccountSessionCookie` 函数
- 在 `src/routes/admin.ts` 中：
  - 新增了 `POST /admin/api/users/:id/impersonate` 路由
  - 新增了 `handleImpersonateUser` 处理函数：验证用户存在且活跃，创建 account session cookie，返回 JSON 及跳转路径
  - 前端用户管理卡片中为活跃用户新增了"登录为"按钮（使用 `data-user-impersonate` 属性）
  - 点击"登录为"时通过 fetch 调用 impersonate API 设置 cookie，然后 `window.open('/account', '_blank')` 在新窗口打开
- Session 签名使用 `USER_CENTER_TOKEN || ADMIN_TOKEN` 作为 secret，确保 admin 登录后即使未单独配置 USER_CENTER_TOKEN 也能正常模拟登录

## 用户创建 Key 支持永久/不限量/不限模型

- 重构了用户中心"创建 API Key"表单 UI：
  - "永不过期"复选框（默认选中），选中时禁用日期选择器
  - "不限模型"复选框（默认选中），选中时禁用模型多选列表
  - 底部 Badge 栏实时显示当前三个维度的状态：用量（不限）、过期（永不/已设定/未设）、模型（全部/已限定）
  - 添加了 `.checkbox-label` 和 `.key-features` CSS 样式
- 提交逻辑根据复选框状态决定 `expires_at`（null）和 `allowed_models`（空数组=不限）
- 用量维度无配额系统，自然不限量
