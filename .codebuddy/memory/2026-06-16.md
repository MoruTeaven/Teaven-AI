## 管理员一键登录其它用户（Impersonate 功能）

- 在 `src/routes/account.ts` 中导出了 `serializeAccountSessionCookie` 函数
- 在 `src/routes/admin.ts` 中：
  - 新增了 `POST /admin/api/users/:id/impersonate` 路由
  - 新增了 `handleImpersonateUser` 处理函数：验证用户存在且活跃，创建 account session cookie，返回 JSON 及跳转路径
  - 前端用户管理卡片中为活跃用户新增了"登录为"按钮（使用 `data-user-impersonate` 属性）
  - 点击"登录为"时通过 fetch 调用 impersonate API 设置 cookie，然后 `window.open('/account', '_blank')` 在新窗口打开
- Session 签名使用 `USER_CENTER_TOKEN || ADMIN_TOKEN` 作为 secret，确保 admin 登录后即使未单独配置 USER_CENTER_TOKEN 也能正常模拟登录

- 重构了用户中心"创建 API Key"表单 UI：

  - "不限模型"复选框（默认选中），选中时禁用模型多选列表
  - 底部 Badge 栏实时显示当前三个维度的状态：用量（不限）、过期（永不/已设定/未设）、模型（全部/已限定）
  - 添加了 `.checkbox-label` 和 `.key-features` CSS 样式
- 提交逻辑根据复选框状态决定 `expires_at`（null）和 `allowed_models`（空数组=不限）
- 用量维度无配额系统，自然不限量

## 新增 /v1/images/generations 图片生成端点

- 问题：后台创建 modality="image" 的模型（如 Kolors）后，调用 `/v1/chat/completions` 被 chat-completions.ts:36 的 `model.modality !== "text"` 校验拒绝
- 修改文件：
  - `src/types.ts` — 新增 `ImageGenerationRequest` 类型（model、prompt、n、size、response_format 等）
  - `src/providers/types.ts` — `ProviderAdapter` 增加 `imageGenerations` 可选方法
  - `src/providers/openai-compatible.ts` — 注册 `images.generations` 能力 + `forwardImageGeneration` 转发到上游 `/images/generations`
  - `src/routes/image-generations.ts` — **新文件**，处理 `POST /v1/images/generations`：校验参数（model/prompt/n/response_format）、检查 modality==="image"、选路由、调 provider
  - `src/routes/chat-completions.ts` — 非 text 模型的错误提示改进：image 模型会提示使用 `/v1/images/generations`
  - `src/index.ts` — 注册 `/v1/images/generations` 路由
- 用法：`POST /v1/images/generations` 传入 `{"model":"Kolors","prompt":"a cute cat","n":1,"size":"1024x1024"}`，携带 Bearer token 认证
